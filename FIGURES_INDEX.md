# STL Figures — Index (two categories)

There are **two kinds** of STL figures. They are **not duplicates of each other** — each figure
shows a **different STL specification / sensor**, and the two categories differ in *how* STL is run.

- **OFFLINE** = STL evaluated on the **recorded 50 Hz dataset** after the flight (`spec.evaluate`),
  attack injected in software. Titles now start with **"OFFLINE STL — …"**.
- **ONLINE** = the **same STL idea run in real time over MAVLink** (`spec.update` per sample),
  with **closed-loop recovery**. Titles now start with **"ONLINE (real-time MAVLink) STL — …"**.

Nothing was deleted. Titles were corrected in-place, and clearly-named copies were grouped into
`figures_offline/` and `figures_online/` (originals `stl_result_*.png` remain in the project root).

## Why there appear to be "many" figures

The 8 OFFLINE figures are **8 different STL properties** (barometer, GPS, gyroscope, a combined
multi-sensor spec, a persistence/temporal spec, two recovery-within-10 s specs, and an altitude
envelope). The 3 ONLINE figures are the **live version** demonstrated on three cases (baro attack,
gyro attack, clean). So the count reflects **distinct specifications and two run modes**, not
repetition of the same result.
